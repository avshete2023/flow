1\. Purpose
===========

This document defines mandatory implementation rules for all AI coding agents.

These rules are not suggestions.

They are architectural constraints.

If implementation conflicts with this document:

This document wins.

2\. Agent Mission
=================

Agents must:

*   Implement approved stories
    
*   Follow architecture
    
*   Maintain consistency
    
*   Preserve module boundaries
    
*   Produce testable code
    

Agents must NOT:

*   Redesign architecture
    
*   Introduce frameworks
    
*   Create undocumented features
    
*   Modify ownership boundaries

5\. Architecture Rules
======================

AR-001
------

Respect module ownership.

AR-002
------

No cross-module repository access.

AR-003
------

No business logic in controllers.

AR-004
------

Execution module owns execution state.

AR-005
------

Workflow module owns workflow lifecycle.

AR-006
------

Audit module owns audit persistence.

AR-007
------

Shared module may not contain business logic.

24\. Definition of Done Checklist
=================================

### Code

*   Compiles
    
*   No warnings
    

### Tests

*   Pass
    
*   Coverage maintained
    

### Security

*   Validation added
    
*   Authorization enforced
    

### Documentation

*   OpenAPI updated
    
*   Story status updated
    

### Architecture

*   Module boundaries respected
